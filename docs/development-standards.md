# 개발 기준 가이드

## 목표

VS Code에서 Quasar 기반 UI 화면을 설계하고, JSON 화면 정의를 Vue 파일로 생성할 수 있는 개발 툴을 구축한다.

## 기본 원칙

- 화면 구조는 JSON 스키마를 기준으로 정의한다.
- 생성 결과물은 Quasar/Vue 표준 구조를 따른다.
- 툴 내부 로직과 생성 대상 애플리케이션 코드는 분리한다.
- 기능은 5단계로 나누어 점진적으로 확장한다.
- xFrame, WebSquare, eXbuilder처럼 화면 정의, 속성 편집, 데이터 연결, 코드 생성 흐름을 명확히 분리한다.

## 기술 기준

- UI 프레임워크: Quasar, Vue
- 개발 도구: VS Code
- 확장 목표: VS Code Extension
- 화면 정의 포맷: JSON
- 생성 대상: `.vue` 파일

## 폴더 기준

- `.vscode/`: VS Code 작업 환경 설정
- `docs/`: 개발/요건/기능 가이드
- `schemas/`: 화면 JSON 스키마
- `src/`: 생성기와 툴 소스
- `templates/`: Vue/Quasar 생성 템플릿
- `examples/`: 샘플 화면 JSON

## 단계별 개발 기준

### 1단계: 화면 JSON -> Vue 파일 생성기

- 화면 JSON 스키마 초안을 정의한다.
- JSON을 파싱하여 Vue SFC 파일을 생성한다.
- Quasar 컴포넌트 매핑 규칙을 문서화한다.
- 생성 결과는 사람이 읽고 수정 가능한 코드여야 한다.

### 2단계: 컴포넌트 팔레트 + 속성 패널

- Quasar 컴포넌트를 팔레트 형태로 분류한다.
- 선택한 컴포넌트의 속성을 패널에서 편집한다.
- 속성 변경은 화면 JSON에 즉시 반영한다.

### 3단계: 드래그앤드롭 화면 배치

- 팔레트의 컴포넌트를 캔버스에 배치한다.
- 배치 순서와 부모-자식 구조를 JSON에 저장한다.
- 레이아웃 이동, 삭제, 복제 기능을 제공한다.

### 4단계: 그리드, 팝업, 공통코드, API 자동연결

- 데이터 그리드 컬럼 정의와 데이터소스 연결을 지원한다.
- 팝업 화면과 호출 관계를 관리한다.
- 공통코드 바인딩 규칙을 정의한다.
- API 명세를 기반으로 화면 이벤트와 데이터 바인딩을 연결한다.

### 5단계: VS Code Extension으로 통합

- VS Code 사이드바 또는 Webview에 UI Builder를 제공한다.
- 프로젝트 파일 탐색, 화면 JSON 편집, Vue 생성 명령을 통합한다.
- Extension 명령 팔레트에서 주요 기능을 실행할 수 있게 한다.

## 코드 작성 기준

- 생성기 로직은 입력 검증, 변환, 파일 출력 단계를 분리한다.
- 템플릿은 하드코딩을 줄이고 재사용 가능한 구조로 유지한다.
- Quasar 컴포넌트 매핑은 설정 파일로 관리한다.
- 오류 메시지는 화면명, 컴포넌트 ID, 속성명을 포함한다.
