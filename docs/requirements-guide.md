# 요건 사항 가이드

## 비전

개발자가 VS Code 안에서 화면을 정의하고, Quasar 기반 Vue 화면 파일을 빠르게 생성할 수 있는 UI 개발 툴을 만든다.

## 사용자

- Quasar/Vue 기반 프론트엔드 개발자
- 업무 화면을 반복적으로 개발하는 SI/업무 시스템 개발자
- 화면 표준화와 코드 생성을 관리하는 아키텍트

## 핵심 요건

- 화면 정의를 JSON으로 저장할 수 있어야 한다.
- JSON 화면 정의를 Vue 파일로 생성할 수 있어야 한다.
- 컴포넌트 선택, 속성 편집, 배치 변경이 가능해야 한다.
- 업무 화면에 자주 필요한 그리드, 팝업, 공통코드, API 연결을 지원해야 한다.
- 최종적으로 VS Code Extension으로 사용할 수 있어야 한다.

## 비기능 요건

- 생성된 코드는 Quasar 프로젝트에서 바로 사용할 수 있어야 한다.
- 화면 JSON은 버전 관리가 가능해야 한다.
- 템플릿과 컴포넌트 매핑은 프로젝트별로 확장 가능해야 한다.
- 기능별 실패 원인을 추적할 수 있는 오류 메시지를 제공해야 한다.
- 대규모 화면에서도 편집 경험이 느려지지 않도록 구조를 설계해야 한다.

## 단계별 요건

### 1단계: 화면 JSON -> Vue파일 생성기

- 화면명, 라우트명, 레이아웃, 컴포넌트 트리를 JSON으로 정의한다.
- JSON 유효성 검사를 수행한다.
- Quasar 컴포넌트 기반 Vue SFC를 생성한다.
- 샘플 JSON과 샘플 생성 결과를 제공한다.

### 2단계: 컴포넌트 팔레트 + 속성 패널

- 컴포넌트를 입력, 표시, 레이아웃, 데이터, 액션 그룹으로 분류한다.
- 컴포넌트별 편집 가능한 속성 목록을 제공한다.
- 속성 변경 시 JSON 모델이 갱신된다.

### 3단계: 드래그앤드롭 화면 배치

- 컴포넌트를 캔버스에 추가, 이동, 삭제할 수 있다.
- 컨테이너 컴포넌트 안에 자식 컴포넌트를 배치할 수 있다.
- 배치 결과는 JSON 컴포넌트 트리로 저장된다.

### 4단계: 그리드, 팝업, 공통코드, API 자동연결

- 그리드 컬럼, 정렬, 포맷, 편집 가능 여부를 정의한다.
- 팝업 화면 정의와 호출 파라미터를 관리한다.
- 공통코드 그룹을 선택 컴포넌트에 바인딩한다.
- API 명세와 화면 이벤트를 연결한다.

### 5단계: VS Code Extension으로 통합

- VS Code 명령 팔레트에서 화면 생성 명령을 실행한다.
- Webview 기반 편집 화면을 제공한다.
- 프로젝트 내 화면 JSON 목록을 탐색한다.
- 생성된 Vue 파일을 지정된 경로에 저장한다.

## 참고 툴에서 적용할 기능

- xFrame: 업무 화면 중심의 빠른 컴포넌트 배치와 데이터 연결
- WebSquare: XML/모델 기반 화면 정의와 속성 중심 편집
- eXbuilder: 화면 빌더, 컴포넌트 팔레트, 이벤트/데이터 바인딩 구조
